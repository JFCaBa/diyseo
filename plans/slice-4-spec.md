# Slice 4 Architecture Plan: AI Article Generation

## 1. Objective
Define the architecture for Slice 4, which introduces an AI-driven article generation pipeline. This engine will use the `BrandProfile` (Brand DNA) as context to generate highly targeted, brand-aligned drafts consisting of a title, excerpt, HTML content, and SEO metadata.

## 2. Scope & Constraints
*   **Included**: Prompt construction using Brand DNA, AI generation flow, internal API endpoint to trigger generation, and database extensions for new fields.
*   **Outputs Required**: `title`, `excerpt`, `contentHtml`, `seoTitle`, `seoDescription`.
*   **Constraints**: Must remain provider-agnostic (designed so OpenAI, Anthropic, etc., can be swapped easily). No changes to the public API from Slice 2. Minimal UI updates (just a trigger button/form).

## 3. Database Schema Updates (Prisma)

We need to extend the `Article` model to accommodate the new output fields generated by the AI.

```prisma
// Updated Article Model
model Article {
  id             String        @id @default(cuid())
  siteProjectId  String
  title          String
  slug           String
  content        String        @db.Text 
  excerpt        String?       @db.Text // New field
  seoTitle       String?       // New field
  seoDescription String?       @db.Text // New field
  status         ArticleStatus @default(DRAFT)
  publishedAt    DateTime?
  createdAt      DateTime      @default(now())
  updatedAt      DateTime      @updatedAt

  siteProject    SiteProject   @relation(fields: [siteProjectId], references: [id], onDelete: Cascade)

  @@unique([siteProjectId, slug])
  @@index([siteProjectId, status])
}
```
*Note: `contentHtml` from the AI output will map to the existing `content` field.*

## 4. Prompt Structure & Context Injection
The core of the generation relies on injecting the structured `BrandProfile` into the system prompt to ensure alignment.

### 4.1. System Prompt Template
```text
You are an expert SEO content writer. Your task is to write an engaging, high-quality article aligned with the following brand guidelines:

--- BRAND DNA ---
Target Audience: {{brand.targetAudience}}
Tone of Voice: {{brand.toneOfVoice}}
Core Values: {{brand.coreValues}}
Mission: {{brand.missionStatement}}
-----------------

Ensure the tone perfectly matches the guidelines above.
```

### 4.2. User Prompt Template
```text
Write a comprehensive article.
Topic/Keyword: {{keyword || "A topic relevant to the brand mission"}}

You MUST output your response as a valid JSON object with the following exact structure. Do not include markdown formatting like ```json.
{
  "title": "A catchy, SEO-friendly title (H1)",
  "excerpt": "A short, engaging 2-3 sentence summary",
  "contentHtml": "The full article content formatted in semantic HTML5 (do not include <html>, <head>, or <body> tags, start with <h2> or <p>)",
  "seoTitle": "Optimized meta title (under 60 characters)",
  "seoDescription": "Optimized meta description (under 160 characters)"
}
```

## 5. API Design & Generation Flow

We will create a new internal Next.js API route or Server Action to handle the generation request.

### 5.1. Internal Generation Endpoint
*   **Endpoint**: `POST /api/internal/sites/:siteId/articles/generate`
*   **Request Body**:
    ```typescript
    {
      keyword?: string; // Optional specific topic to guide the generation
    }
    ```

### 5.2. Generation Workflow
1.  **Request Received**: Client triggers generation with an optional keyword.
2.  **Context Assembly**: Backend fetches the `BrandProfile` associated with `siteId`.
3.  **Prompt Construction**: Injects Brand DNA and keyword into the prompt templates.
4.  **AI Invocation**: Calls the AI provider wrapper (returning a JSON string).
5.  **Parsing & Validation**: Parses the JSON and validates it against the `GeneratedArticleSchema`.
6.  **Slug Generation**: Creates a URL-safe `slug` from the generated `title` and ensures uniqueness.
7.  **Database Save**: Creates a new `Article` record with `status: "DRAFT"`.
8.  **Response**: Returns the created `Article` ID so the UI can redirect the user to the editor.

## 6. Validation Rules (Zod)

```typescript
import { z } from 'zod';

export const GenerateArticleRequestSchema = z.object({
  keyword: z.string().max(100).optional(),
});

export const GeneratedArticleSchema = z.object({
  title: z.string().min(1),
  excerpt: z.string().min(1),
  contentHtml: z.string().min(1),
  seoTitle: z.string().max(60),
  seoDescription: z.string().max(160),
});
```

## 7. AI Service Abstraction (Extensibility)
To remain provider-agnostic, the generation logic should be abstracted behind an interface:

```typescript
interface AIGenerationService {
  generateArticle(prompt: string, systemContext: string): Promise<GeneratedArticleSchema>;
}
// Implementations: OpenAIGenerator, GeminiGenerator, etc.
```

## 8. Acceptance Criteria
*   [ ] `Article` database model is updated with `excerpt`, `seoTitle`, and `seoDescription` fields.
*   [ ] An internal API endpoint or Server Action exists to trigger generation for a specific site.
*   [ ] The generation logic successfully fetches and injects the site's `BrandProfile` into the prompt.
*   [ ] The AI output is strictly validated against the expected JSON schema.
*   [ ] Successfully generated content is automatically saved to the database as a `DRAFT` article.
*   [ ] The generation service architecture allows swapping the underlying AI provider without changing the routing or database logic.
